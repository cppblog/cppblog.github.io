因为运算越界导致的问题，几乎所有的语言都能遇到。Python中float提供大约17位的精度，超过这个长度运算就会出问题。 在python中如下图：

输出的结果并不是原始数据值。 所有的语言都能遇到这样的问题，看下面这个C代码。

#include<stdio.h>
#define MAX 255
int main()
{
	unsigned char ARRAY[257], i;
	for (i = 0; i <= MAX; i++)
	{
		ARRAY[i] = i;
		printf("i = %d\n",i);
	}
	return 0;
}

这个代码乍一看没什么问题，当执行到ARRAY [255] = 255,之后会执行i++，此时i会从255变成0，为什么呢？就是因为数据类型，unsigned char的范围是[0,255],当变量达到255时，继续执行就会数值越界产生溢出，就会变成0，这段看似没有问题的代码最终会无限循环下去。所以程序员无论用什么语言在开发的过程中，一定要注意数据类型。
