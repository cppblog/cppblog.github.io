一种无锁队列的实现方法
Posted on Nov 18, 2021 by 陽光不鏽
编程

现代开发语言都提供队列数据结构，如在C++中stl容器队列为queue,另外还有一些用户自定义衍生品实现，通过链表或者数组实现的队列，但他们大多数是非线程安全的，即使提供的线程安全的队列，由于使用了锁，在高性能领域加锁的互斥会影响性能，这里介绍一种无锁的队列，作一个简单的分享总结。

0x00 概述

本文系参考了Linux内核的队列kfifo,其是一个轻量且非常巧妙的设计实现，它本质是一个具有限定大小的buffer,在内核级生产消费环境中无锁，设计巧夺天工，性能强悍，具体关于系统内核kfifo的分析资料有很多，此处不再赘述，本文的无锁队列采用kfifo的思想，采用C++语言级别通过内存栅栏（也称内存屏障）来实现。

0x01 相关主题

  • 1.1 并发
    并发影响着计算机系统的设计，人类对速度的追求是无止境的，为了提高运行速度，处理器一直在提高处理频率，但这会带来更大的功耗，为了避免物理上的限制而保持高性能，处理器开始采用多核设计，各个处理器协同工作完成任务，这样处理器就可以在较低频率下运行，但是多核对于无法拆分的任务处理是不理想的，而且任务之间需要内核进行通信，这种通信的成本也限制着多核系统的性能。市面上主流的处理器在多核之间的通信是利用共享内存来实现的，通过对内存一读一写，相比计算速度，通过内存存取是相当缓慢的。所以在处理器设计上又通过引入缓存和缓冲区来减少内存延迟。在这种存储器的设计中，有两个方案：1.设计复杂的协议来隐藏细节，保持通信低延迟且封装出能对外提供简单的接口；2.在机器内部的处理上让存储器乱序访问来减少延迟。目前主流处理器厂商设计均选择后者方案。

CPU架构的演进

  • 1.2 内存模型
    1.2.1 关于c++11内存模型
    关于内存模型最原始的需求在于操作系统内核，大多数编程时程序员不需要直接面向它，随着时代的发展，出于多核并行任务的需求，CPU本身的缓存，编译器优化、CPU体系指令乱序等原因，从而在应用层面得到越来越多的关注。

    不同CPU体系结构的内存顺序模型是不一样的，如下表格所描述:

      |       arch           | Memory order              |
  | -------------------- | ------------------------- |
  | x86_64               | Total Store Order         |
  | Sparc                | Total Store Order         |
  | PowerPC              | Weakly Ordered            |
  | ARMv8                | Weakly Ordered            |

    其次不同的编程语言对内存模型也有自己的规范，C11/C++ 11 ISO Standard的发布，带来memory order的标准支持。Bjarne老爷子在HOPL4论文中有描述，C++ 11模型基于先行发生（happens-before）关系确定顺序，既支持宽松的内存模型，也支持顺序一致，除此之外C++11还提供了对原子类型和无锁编程的支持，并且与之集成。

    节选Bjarne HOLP4论文原话：
    One of the hardest decisions was to accept both Intel’s x86 primitives (a Total Store Order (TSO) model [TSO Wikipedia 2020] plus a few atomic operations) and IBM’s PowerPC primitives (weak consistency plus fences) for the lowest level synchronization. Logically, only one set of primitives were needed,but Paul Mckenney convinced me that there was far too much code written using fences deepin complex algorithms for IBM to adopt something like Intel’s model. One day, I literally didshuttle diplomacy between two corners of a large room. Eventually, I suggested that both had to be supported and that was what C++11 adopted.

    1.2.2 原子操作的内存序

      /// Enumeration for memory_order
  typedef enum memory_order
  {
    memory_order_relaxed,
    memory_order_consume,
    memory_order_acquire,
    memory_order_release,
    memory_order_acq_rel,
    memory_order_seq_cst
  } memory_order;

    默认内存序列都是memory_order_seq_cst,以上六个选项中，代表三种内存模型：顺序一致性(sequentially consistent)，获取-释放序(memory_order_consume, memory_order_acquire,memory_order_release和memory_order_acq_rel)和自由序(memory_order_relaxed)。我们关注自由序的加强版–获取-释放序，这种序列模型中，原子加载获取(acquire)操作(memory_order_acquire)，原子存储就是 释放(memory_order_release)操作。

    1.2.3 内存栅栏

    内存栅栏操作会约束内存序列，冻结数据修改，通常与memory_order的原子操作一起使用，栅栏属于全局操作，执行栅栏操作会影响到在线程中的其他原子操作，栅栏就是一道无法跨越的鸿沟，所以内存栅栏操作通常也被称为内存屏障(memory barriers)。我们给环形队列在原子操作中添加一个栅栏，栅栏可以阻止无序让自由操作变的有序。

    1.2.3 内存模型的作用和意义
    内存模型引入的目的是建立一个统一的标准，我们可以从语言的层面使用内存序来实现与运行平台和编译器无关的多线程下的交互控制,从语言的级别实现跨平台。

  • 1.3 无锁编程
    无锁编程是编写不显式使用锁的并发程序技术，程序员依靠原语操作（由硬件直接支持）来避免小对象的数据竞争，不必忍受数据竞争的原语操作被称为原子操作。详见 Bjarne 2016 [2]

  • 1.4 关于环形队列
    环形缓冲队列是提供了一个无边界的字节容器，当其中的元素被消费不影响整个队列的存储，也就是说它没有移动和拷贝的性能损失，且是无锁的队列。

  • 2.1 实现原理
    根据内核kfifo
    1.初始化的空间为2的次幂（2次幂优点不再赘述） 2.无符号整数保存in和out，不做取模运算，让其自然溢出
    3.使用内存栅栏

  • 2.2 功能特点
    1.单生产、消费实现无锁访问
    2.因为是环形队列，可能会被覆盖，如果队列缓冲已满，进行阻塞。

  • 2.3 代码实现

      class FifoBuffer {
  public:
      FifoBuffer(){};
      virtual ~FifoBuffer(){};
    
      auto put(const uint8_t *data, uint32_t len) -> void
      {
          uint32_t lp = 0;
          while(len>(maxsize_ - in_ + out_))
              std::this_thread::yield();
          lp = std::min<uint32_t>(len, (maxsize_ - (in_ & (maxsize_ - 1))));
          memcpy(data_ + (in_ & (maxsize_ - 1)), data, lp);
          memcpy(data_, data + lp, len - lp);
          in_ += len;
          std::atomic_thread_fence(std::memory_order_release);
      }
    
      auto get(uint8_t *to, uint32_t len) -> uint32_t
      {
          uint32_t lg = 0;
          len = std::min(len, in_ - out_);
          lg = std::min(len, maxsize_ - (out_ & (maxsize_ - 1)));
          memcpy(to, data_ + (out_ & (maxsize_ - 1)), lg);
          memcpy(to + lg, data_, len - lg);
          out_ += len;
          std::atomic_thread_fence(std::memory_order_release);
          return len;
      }
      inline auto getDataLength() const -> uint32_t{
          std::atomic_thread_fence(std::memory_order_acquire);
          return  in_ - out_;
      }
  private:
      constexpr static const uint32_t maxsize_ = 1 << 20;//1MB
      int8_t data_[maxsize_] {0};
      uint32_t in_ = 0;
      uint32_t out_ = 0;
  };

0x03 结果分析

150字节字符，循环100万次 put std::queue : 45411466 ns get std::queue : 14296225 ns put fifobuf : 30156793 ns get fifobuf : 89319 ns

注:时间越少性能越好
由于写操作是一次一次的写入，跟标准库性能差不太多，读操作环形缓冲是可以一次性读多个，且数据不随读取拷贝和移动，所以性能有100多倍的提升。
